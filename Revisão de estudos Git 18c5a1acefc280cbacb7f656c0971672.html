FLUXO BÁSICO DE TRABALHO COM GIT

  1. Cria diretório;
  1. Inicia o git dentro do diretório que foi criado anteriormente → git init .
  1. Edita ou cria os arquivos
  1. Valida se o status da alterações → git status
  1. Por vias de boa prática, crie uma branch alternativa utilizando → git checkout -b “nome_branch_alt”, dessa forma podemos fazer commits para um branch alternativa e só depois avaliar através do PR se podemos levar as alterações para a branch main.
  1. Adiciona os arquivos a fila de commit → git add <nome_do_arquivo>
  1. Valida novamente se efetuou a adição anterior → git status
  1. Efetua o commit da fila de commit organizada → git commit -m “nome_do_commit”;
  1. Valida novamente se o commit foi corretamente → git status
  1. Caso tenha um repositório local e queira conectar com um repositório remoto (GitHub por exemplo), use o comando → git remote add origin <link do repositorio>
  1. Em seguida após conecta-lo faça o push para a branch alternativa → git push -u origin <nome da branch secundaria>
  1. Verifique a alteração no seu diretório GitHub ou outro que estiver utilizando, e execute o ‘compare & pull request’ o famoso PR
  1. Após criar o pull request no GitHub, tendo feito tudo corretamente precisará apenas executar um merge do PR e seus últimos ajustes no código estarão devidamente versionados na sua main branch do seu projet

RESUMO GERAL DE COMANDOS GIT

  1. O que é Commit?
    1. Fazendo uma analogia é como se fosse um save progress ou checkpoint. Nos quais você pode retornar a cada um desses checkpoints quando achar necessário. Comitou? Criou um checkpoint e assim por diante. Dessa forma, criando versões do código ou seja, o versionamento.
  1. Comandos básicos:
    1. cd → Muda de diretório, exemplo: cd Desktop/ ← muda o diretório para o caminho da pasta Desktop;
    1. ls → Lista os itens contidos dentro do diretório que foi determinado, como por exemplo o desktop;
    1. pwd → Retorna o caminho completo de onde você está (print work director);
    1. ls -a → Mostra todos os itens contidos incluindo os ocultos;
    1. cd .. → Desloca para a pasta anterior de onde você está atualmente;
    1. cd . → É a própria pasta onde você está;
    1. mkdir → Cria um nova pasta chamada ‘nome_pasta’, exemplo: mkdir nome_pasta
    1. cat → Exibe o que tem contido dentro de qualquer arquivo editável, exemplo: cat nome_arquivo.txt
    1. git init . → cria um repositório git vazio dentro do caminho
    1. rm -rf . git/ → apaga a pasta/repositório vazio caso haja alguma erro, etc.
      1. -rf é um acrescimo de comando que força apagar sem nenhum tipo de interrupção, ou seja, rf = recursão force. Ou seja, cuidado isso pode apagar qualquer coisa sem ter o que fazer. * CUIDADO AO UTILIZAR*
    1. git status → Dá um resumo geral da branch utilizada
      1. untracked files → são arquivos que ainda não foram comitados ou seja não há versionamento.
    1. git add → Adiciona o arquivo que eu quero versionar com o commit. Exemplo: git add nome_arquivo.txt , aqui você adiciona esse arquivo no commit.
    1. git commit -m → Fazer um commit e nomea-lo por exemplo: git commit -m “comandos bash”
    1. git config —global user.email “email@.com.br” - > assim você seta qual email/user sera definido para todos os repositórios.
    1. git config —global user.email ou git config —global user.name → verifica qual email ou nome está setado atualmente.
    1. git log → Verifica os logs de todos os commits já feitos
    1. . → diretório atual
    1. .. → diretório anterior ao atual
    1. * → se refere a tudo. Por exemplo: ‘git add comandos_*’ , esse comando adicionaria para fila de commit todo os arquivos dentro ta pasta atual q tenham nome q contenham ‘comandos_’.
    1. git diff → mostra a diferença ou alterações entre commits, por exemplo: ‘git diff comandos_git.txt’
    1. git reset → Altera o status de um arquivo de staged para unstaged, ou seja, remove ele da fila de add para o proximo commit. Exemplo: git reset ‘nome_arquivo.txt’
    1. git checkout -b → Cria uma branch ou ramificação da branch main ou principal, exemplo: git checkout -b ‘nome_branch_secundaria’.
      p.s: Utiliza-se o termo -b é só quando a branch não existe ainda.
    1. git branch → Exibe quais as branchs existentes e qual branch você está no momento.
    1. git merge → Unifica as duas branchs, exemplo: git merge secundaria.
      1. Vale notar que, se fizemos merge de uma branch secundaria_1 com a main e depois um merge de outra branch secundaria_2 também com a main, tanto a branch secundaria_1 como a branch secundaria_2 estarão inalteradas, pois só fizemos as alterações presentes em cada branch para main e não unindo as duas branchs secundarias antes de mergear com a main.
      1. a ideia tb é sempre sua branch pessoal fazer merge com a main pra levar as alteracoes da SUA BRANCH, entao caso outra pessoa faça um merge entre a branch dela com a nova main branch (alterada pelo seu avanço) pode assim ocorrer um conflito.
    1. git branch -d <nome_branch> → Apaga a branch selecionada.
    1. git remote add origin → Conecta o repositório local com o repositório remoto no GitHub, Gitlab, etc.
    1. git remote remove origin → remove o link com repo pra vc trocar de projeto por exemplo
    1. git push - u origin <branch que quer enviar a alteração> → Esse comando envia todas as alterações da branch local da main para branch remota <origin>. Ou seja, envia as coisas alteradas no ultim commit para a repo remota.
      1. GIT PUSH ENVIA ALTERACOES DE X PARA, EXEMPLO: origin main, quer dizer que está enviando da main (local) para remota (origin).
    1. touch <nome do arquivo> → criar uma arquivo vazio
    1. git pull - u origin <branch que quer puxar a alteração> → então você quer RECEBER alterações da origin.
    1. git clone <link do repositório> → clona todo o repositório para sua máquina local.
